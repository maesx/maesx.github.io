# 实例分割可视化说明

## 🎯 功能说明

实例分割（Instance Segmentation）能够**区分同一类型的不同个体**，例如：
- 两辆轿车分别用**蓝色**和**橙色**标注
- 三辆卡车分别用**绿色**、**紫色**、**黄色**标注
- 每个独立车辆都有唯一的编号和颜色

1. **不同颜色**: 每个实例用不同颜色

## 🔬 技术原理

### 语义分割 vs 实例分割

**语义分割（Semantic Segmentation）**:
- 只区分类别：所有轿车都是同一个颜色
- 无法区分个体：两辆轿车会被标记为相同颜色

**实例分割（Instance Segmentation）**:
- 区分个体：每辆轿车都有独立颜色
- 额外信息：可以统计车辆数量、测量面积

### 实现方法：

1. **语义分割预测** → 使用训练好的模型
2. **连通区域分析** → 分离独立目标
3. **噪声过滤** → 过滤小于100像素的区域
4. **实例编号** → 为每个实例分配唯一ID
5. **颜色映射** → 为每个实例分配独特颜色

## 📞 问题排查

**Q: 为什么有些小区域没有标注？**
A: 系统会过滤小于100像素的区域，这些通常是噪声或分割错误。

**Q: 实例颜色是如何生成的？**
A: 使用预设的20种高对比度颜色循环分配，确保相邻实例颜色差异明显。

**Q: 可以修改最小区域阈值吗？**
A: 可以，在代码中修改 `if area < 100` 这一行。
